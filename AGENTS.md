# WitchHunt — 자동화 에이전트 지침

이 파일은 이 저장소에서 작업하는 에이전트(코덱스 등)가 **매번 따라야 하는 규칙**이다.
특히 **뉴스 자동 갱신 파이프라인에서 `_summaries.json`을 만들 때** 아래를 반드시 지킨다.

## `_summaries.json` 작성 규칙 (중요)

`_articles.json`을 읽고 각 `id`마다 아래 필드를 만들어 `_summaries.json`에 저장한다.
형태: `{ "기사id": { ...필드... }, ... }` — 순수 JSON 객체, UTF-8, 코드펜스 금지.

### 기본 필드
- `summary` — 사실 위주 한국어 2~3문장 요약. **반드시 완결된 문장으로 끝낼 것**(`~다.` 등).
  기자 이름·사진 캡션·소제목·`&middot;` 같은 원문 조각을 그대로 쓰지 말 것.
- `background` — 배경·맥락 1~2문장.
- `importance` — 1~10 정수.
- `category` — `정치` `경제` `사회` `국제` `주식` `크립토` `예측시장` 중 하나.
- `publicTake` — (댓글 자료가 있을 때만) 네티즌 반응 한 문장.

### ★ 진영별 주장 필드 — `issue` / `viewLeft` / `viewRight` (놓치지 말 것)

`_articles.json`의 항목에 **`views` 자료가 들어 있으면**, 그 항목에는
`views.left`(더 진보 쪽 진영)와 `views.right`(더 보수 쪽 진영)의
**실제 기사 본문(text)·매체명(outlet)·진영(lean)** 이 담겨 있다.

이 두 본문을 읽고, **"진보 진영과 보수 진영이 이 사건을 두고 각각 무엇을 주장하는지"** 를 정리해서
`issue`, `viewLeft`, `viewRight` 세 필드를 만든다. (특정 매체 한 곳이 아니라 **그 진영 전체의 주장**을 대표하도록 쓴다.)

- `issue` — **쟁점 한 줄**. 두 진영이 '무엇을 두고' 갈리는지 한 구절로. (예: `"특검의 잇단 영장 기각을 어떻게 볼 것인가"`)
- `viewLeft` — **진보 진영의 주장** 1~2문장.
- `viewRight` — **보수 진영의 주장** 1~2문장.

작성 규칙:
- **"이 진영은 무엇을 강조하고, 무엇을 문제 삼으며, 어떻게 주장하는가"** 를 드러낼 것. 단순 사실 반복 금지.
  - 좋은 예: `"정황이 뚜렷한데 법원이 지나치게 신중하다며, 수사가 위축되면 안 된다고 본다."`
  - 나쁜 예: `"이 사건에 대해 보도했다."` (주장이 아님)
- 그 진영 기사가 **실제로 담고 있는 내용**에 근거할 것. 없는 주장을 지어내지 말 것.
- `viewLeft`와 `viewRight`는 **서로 무엇이 다른 주장인지 대비되게** 쓸 것. 둘이 똑같으면 안 된다.
- 매체명·"진보 매체는"·"한겨레는" 같은 말머리는 **넣지 말 것**(화면이 '진보 진영'으로 표시함). 바로 주장 내용부터.
- 완결된 문장으로 끝낼 것. 욕설·단정적 허위·특정인 비방 금지. 진영을 조롱·비하하지 말 것(중립적 서술).

`views` 자료가 없는 항목에는 `issue`/`viewLeft`/`viewRight`를 넣지 않는다.

### 예시
```json
{
  "p1-c0": {
    "summary": "...",
    "background": "...",
    "importance": 8,
    "category": "정치",
    "issue": "특검의 잇단 구속영장 기각을 어떻게 볼 것인가",
    "viewLeft": "정황이 뚜렷한데 법원이 지나치게 신중하다며, 수사가 위축돼선 안 된다고 본다.",
    "viewRight": "무리한 별건·먼지털이 수사의 결과라며, 영장 기각이 곧 수사의 정당성 문제라고 본다."
  }
}
```

## 그 밖의 규칙
- 한글이 `?`나 `�`로 깨지지 않게 **UTF-8**로 저장하고, 저장 후 다시 읽어 확인할 것.
- 뉴스 화면 디자인은 **무채색 + 성향색(진보 파랑/보수 빨강)만** 사용한다.
- 네트워크가 필요한 수집 스크립트(`build-feed`·`dump-articles`·`fetch-reactions`)는
  일시적 실패에 대비해 재시도가 들어 있다(`scripts/lib-fetch.mjs`). 실패 로그가 보이면 원인을 기록할 것.
