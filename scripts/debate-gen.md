# AI 토론 생성 지침 (debates.json)

이 파일은 WitchHunt 'AI 토론'을 생성할 때 따르는 규칙이다. 저장소 안 파일만 읽고 `public/debates.json`을 새로 덮어쓴다. 코드 파일은 수정하지 않는다. 네트워크/웹검색 금지.

## 입력
- `scripts/personas.json` — 페르소나 12명: id, name, initial, role, bio, speech(말투 지침), topics, conflicts.
- `public/feed.json` — events: id, category, title, summary, background, importance, outletCount.

## 고를 사건 (10개, 분야 균형)
피드에서 사건 10개를 골라 각각 토론 스레드를 만든다.
- 7개 분야(정치·경제·사회·국제·주식·크립토·예측시장)를 각각 최소 1개씩 먼저 채운다(그 분야 사건이 피드에 있으면).
- 남는 자리는 보도량·중요도 높은 분야(보통 정치·경제·국제)에서 1~2개 더 뽑는다.
- 같은 사건을 두 스레드로 만들지 않는다(eventId 중복 금지).

## 각 스레드
- 참여(댓글): 그 category가 topics에 든 의견형 페르소나 4~5명. ai_012(쟁점정리)는 댓글에 넣지 않는다. 시각이 갈리도록 conflicts 쌍을 최소 한 쌍 포함.
- messages 8~11개: 1라운드 각자 관점 → 2~3라운드 서로 받아치기·동의 섞기. 받아치는 발언은 replyTo에 상대 message id를 넣는다.
- pointSummary: ai_012(쟁점정리)가 존댓말로 핵심 쟁점 2~3개를 짧게(3~4문장) 정리. 매번 다른 어투로.

## 말투 (가장 중요 — 사람 커뮤니티처럼)
- 각 발언은 그 페르소나의 speech를 따른다. 반말 캐릭은 반말, 존댓말 캐릭은 존댓말로 일관되게.
- 구어체로 짧고 톡톡(보통 1~2문장). 가끔 아주 짧은 잽·맞장구도 섞는다("음, 글쎄.", "그건 인정.", "ㄹㅇ 그 부분은 맞지.").
- speech의 예시 문구를 그대로 복붙하지 않는다(말투·성향만 참고, 매번 새 표현). 같은 페르소나가 여러 스레드에서 같은 첫마디를 반복하지 않는다.
- 반박을 매번 "{닉네임}," 로 시작하지 않는다. 이름을 문장 중간에 넣거나 생략 등 다양하게.
- 항상 반대만 하지 말고, 가끔 동의·편들기에 한 가지 덧붙이는 식도 섞는다.
- 'ㅋㅋ/ㅋㅋㅋ'는 가벼운 반말 캐릭(테크스파크·의심회로·마켓펄스·메타렌즈)이 어울리는 순간에만 가끔 쓴다(매 발언 X). 팩트체크중은 가끔 건조한 "ㅋ" 정도. 진지한 존댓말 캐릭(휴먼코스트·브레이크포인트·넘버스택·롱뷰·프로토콜·북극성경계·쟁점정리)은 ㅋㅋ를 쓰지 않는다.

## 금지
- 욕설·혐오·특정집단 비하·폭력 선동.
- 허위사실 단정(불확실하면 "~인 듯", "~라는 말 있던데").
- 투자 수익 보장 표현, 숫자 지어내기.
- 내부 데이터 필드명(outletCount, importance, eventId, category 등)을 발언 본문에 절대 노출하지 않는다. 넘버스택도 그런 메타 숫자 말고 기사 속 실제 내용/일반적 추세로 말한다.
- 짧아도 알맹이는 있게(빈 리액션만 늘리지 않기).

## 출력: public/debates.json (순수 JSON, UTF-8, 덮어쓰기)
```
{
  "generatedAt": "<현재 ISO 시각>",
  "personas": [ { "id","name","initial","role","bio","topics","conflicts" }, ... personas.json의 12명 전부(ai_012 포함, 이 7필드만. topics·conflicts는 그대로 복사) ],
  "threads": [
    { "id":"th1", "eventId":"<사건 id>", "category":"<category>", "title":"<사건을 질문형 쟁점으로>",
      "issueSummary":"<무슨 일인지 1~2문장>",
      "messages":[ {"id":"th1-m1","personaId":"ai_003","replyTo":null,"text":"..."} ],  // ai_012 없음
      "pointSummary":"<ai_012의 정리 3~4문장>" }
  ]
}
```
- threads 10개. message id는 `th{n}-m{k}`(th1~th10). 코드펜스·설명 문구 금지, `{`로 시작 `}`로 끝나는 JSON만 파일에 쓴다.
- ⚠️ 반드시 UTF-8로 저장하고, 저장 후 다시 읽어 한글이 `?`로 깨지지 않았는지 확인한다.
